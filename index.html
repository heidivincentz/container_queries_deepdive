CSS Container Queries

A lot of developers seem excited

When you take a look around the internet, it would seem that this layout tool has been requested by the many for quite some time.

Container queries could be a new and easier way to set up responsive layouts, being much more adaptible across all sreen sizes.

What makes container queries different from media queries?

Container queries allows you to adjust the layout according to the space the container (that you define) has available. Instead of based on the space that's available in the viewport.

So where media container queries come in handy when you want to be 'changing the style of an element based on the size of one of its nearest containers'

It would seem that container queries can actually replace grid and flexbox , but it is still recommended to continue using grid and flexbox as a fallback for browers, that do not support container queries.

When will it be available?

CSS container queries tool is still in the early stage of development. I haven't been able to find any information on when they are expecting for a more browsers to be supporting this. For now it is only available for trying out in Google Canary.

How to set up Google Canary

  1. First download Google Canary.
  2. Go to chrome://flags/.
  3. Search for container.
  4. Enable Enable CSS Container Queries by switching it from default to enabled. It's going to ask you to restart the browser.
  5. Start the live view up in VS Code.
  6. Copy the IP address that opens up in your default browser and paste it into Google Canary.
